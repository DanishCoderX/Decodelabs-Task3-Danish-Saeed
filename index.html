Tab Switcher

The Document Object Model

The DOM is the browser's living map of your HTML. It translates static markup into a tree of objects that JavaScript can read and mutate. Every tag becomes a node; every node is addressable.

const el = document.querySelector('.js-hook');

Event Listeners

The browser listens for stimuli — clicks, keypresses, and system triggers. addEventListener wires a specific node to a handler function that fires when the event occurs.

btn.addEventListener('click', handleClick);

State Management

State is the collection of data that determines what the user sees. Use const for DOM references that never reassign, and let for values that must mutate. Never use var.

let count = 0;  // mutable state

Feature 02

Accordion FAQ

Each item uses classList.toggle('is-open'). JavaScript controls the behavior; CSS handles the animation.

Every interactive element follows the Input–Process–Output loop. The Input is the user event (a click). The Process is the JavaScript logic that evaluates state. The Output is the visible DOM mutation that results.

innerHTML parses and renders HTML, which opens a Cross-Site Scripting (XSS) vulnerability if the content comes from user input. textContent treats everything as plain text — it is safe by default.

js- prefixed classes are JavaScript hooks only — never styled with CSS. is- prefixed classes define visual state (e.g. .is-active, .is-open) and are toggled by JS but styled by CSS. This decoupling makes code robust and maintainable.

const creates an immutable binding — use it for DOM references and values that never reassign. let is block-scoped and mutable — use it strictly when the value must change, such as a counter. var is legacy and should be avoided entirely.
